# Icon

[Material Icons][1] with the base class `ada-icon`.

Subcomponents **cannot** have the `children` property overridden.

**Accessibility**

An `Icon` can have 2 purposes:

1. **decorative only**: It illustrates the label next to it. To ensure the icon
is ignored by screen readers, set `aria-hidden` to `true`.
  - Ex: `<ArrowUpwardIcon aria-hidden />`

1. **non-decorative**: Used to present information, such as an `Icon` element
being the only child of a `Button` component. The icon needs to be labelled
appropriately.
  - Ex: `<ArrowUpwardIcon aria-label='Scroll to top' />`

[1]: https://material.io/resources/icons/?style=outline
